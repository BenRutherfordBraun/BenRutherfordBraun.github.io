Rear Sensor Module (RSM)

High-speed data acquisition and CAN-Bus integration for vehicle dynamics.

Project Definition

The Rear Sensor Module (RSM) is a critical node in the car's control network. It is responsible for monitoring real-time telemetry from the rear suspension and drivetrain to optimize torque vectoring and traction control algorithms.

Interactive 3D Render of the RSM v2.1 (UBC Formula Electric 2025)

Technical Challenges

  • EMI Mitigation: Designing for a high-noise environment near the electric motors and inverters.
  • CAN-Bus Reliability: Implementing a robust protocol to ensure zero packet loss during high-G maneuvers.
  • Real-Time Processing: Handling interrupt-driven sensor sampling with sub-millisecond latency.

C++ Firmware Snippet

Below is the implementation of the CAN message packing for the RSM telemetry frame:


// RSM Telemetry Frame Construction
void RSM_Send_Status() {
    CAN_Message_t msg;
    msg.id = CAN_ID_RSM_STATUS;
    msg.len = 8;

    // Pack 12-bit sensor data into 8-byte payload
    msg.data[0] = (uint8_t)(suspension_travel >> 4);
    msg.data[1] = (uint8_t)((suspension_travel << 4) | (temp_sensor >> 8));
    msg.data[2] = (uint8_t)(temp_sensor & 0xFF);

    CAN_Transmit(&msg);
}

Specifications

  • Microcontroller: STM32F4 Series
  • Communication: CAN 2.0B @ 1Mbps
  • Inputs: 4x Analog, 2x Digital Hall Effect
  • Power: 12V DC (Isolated)
